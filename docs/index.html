プロジェクトを始めるたびに
同じ作業を繰り返すのを、やめる。

ウィンドウ分割、docker compose up、npm run dev、AI エージェントの起動——
プロジェクトを開くたびに、同じ手順を繰り返していませんか？
muxflow はプロジェクトごとの起動手順を記録するターミナルのプロジェクト管理ツールです。
登録したら次回から Enter 1回で、全部が立ち上がります。

// the problem

開発を始めるまでに、やることが多すぎる。

// 毎回同じ環境を手動で再構築
tmux split-window -h tmux send-keys "npm run dev" tmux split-window -v tmux send-keys "docker compose up"

再起動のたびに、同じペイン分割と同じコマンドを繰り返す。頭の中では作業がはじまっているのに。

// AI エージェントの起動も増えた
claude --project myapp # + MCP サーバーの起動 # + コンテキストの再設定 # まだ作業ゼロ...

AI エージェント時代になって、プロジェクト開始前にやることがさらに増えた。

// 複数プロジェクトの把握
$ tmux ls proj-a: 2 windows proj-b: 1 window (detached) temp-1: 1 window what-was-this: 1 window

並行開発が増えるほど、どのセッションがどのプロジェクトか把握できなくなる。

// how muxflow helps

プロジェクトの起動を、一度だけ設定する。

01

プロジェクト一覧から
起動・切り替え・管理。

Git リポジトリを自動スキャンして一覧化。
どのプロジェクトが今動いているか一目で把握でき、Enter 1回で起動・アタッチ・切り替えができます。
複数プロジェクトを並行開発しているときのプロジェクト管理がシンプルになります。

s スキャン  →  Space 選択  →  Enter 起動
▶ demo-scan.gif // GIF がここに入ります
02

レイアウトとコマンドを登録して、
毎回 Enter だけ。

ウィンドウ分割・ペイン構成・起動コマンドをプロジェクトごとに登録。
docker compose up、npm run dev、AI エージェントの起動コマンドもまとめて設定できます。
次回から Enter 1回で、全部が同時に立ち上がります。

e エディタを開く  →  コマンド登録  →  w 保存
▶ demo-editor.gif // GIF がここに入ります
03

★ マークで、
再起動後も即座に復旧。

a キーでプロジェクトに ★ を付けておくと、tmux セッションがゼロの状態で起動したとき自動で一括起動します。
PC を再起動しても、A を押すだけでレイアウトもコマンドも全部戻ってくる。
「環境を再構築する時間」が、まるごとなくなります。

a 自動起動フラグ  →  A 一括起動
▶ demo-autostart.gif // GIF がここに入ります
// key bindings

覚えるキーは、少ない。

Enter セッション起動、または起動中ならアタッチ
j / k カーソル上下移動
e ウィンドウ・ペイン編集画面を開く
s Git リポジトリをスキャンしてプロジェクト追加
a 自動起動フラグのトグル（★）
A ★ のついた全プロジェクトを一括起動
x セッション停止
n 新規セッション作成（クイックスタート）
X プロジェクトを非表示に移動 / 復元
K / J プロジェクトの並び順を変更
? ヘルプオーバーレイを表示
q 終了

// アプリ内で ? を押すとフルキーバインド一覧が表示されます

// install

30秒ではじめる。

  1. 1
    tmux をインストール
    $sudo apt install tmux
    $brew install tmux
  2. 2
    muxflow をインストール
    $curl -L https://github.com/masaway/muxflow/releases/latest/download/muxflow_linux_amd64.tar.gz | tar xz && mv muxflow ~/.local/bin/
    $curl -L https://github.com/masaway/muxflow/releases/latest/download/muxflow_linux_arm64.tar.gz | tar xz && mv muxflow ~/.local/bin/
    $curl -L https://github.com/masaway/muxflow/releases/latest/download/muxflow_darwin_arm64.tar.gz | tar xz && mv muxflow /usr/local/bin/
    $curl -L https://github.com/masaway/muxflow/releases/latest/download/muxflow_darwin_amd64.tar.gz | tar xz && mv muxflow /usr/local/bin/
    $go install github.com/masaway/muxflow@latest

    // Go 1.24.2 以上が必要です

  3. 3
    起動する
    $muxflow

    // 初回起動時はスキャンディレクトリの設定画面が自動で開きます（例: ~/work）
